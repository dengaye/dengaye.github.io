mixins使用

发表于 2019-07-04

在一些大型的项目中，经常会遇到一些组件使用相同功能的情况。如果每个组件都假如相同的代码，代码维护性就会变得越来越到，mixins就是为了解决这一情况。

在vue中的使用

官方文档

使用

合并规则

当组件和混入对象含有同名选项时，这些选项将以恰当的方式进行“合并”

  • 值为对象的选项，例如 methods、components 和 directives，将被合并为同一个对象。两个对象键名冲突时，取组件对象的键值对。

  • 数据对象在内部会进行递归合并，并在发生冲突时以组件数据优先。

  • 同名钩子函数将合并为一个数组，因此都将被调用。另外，混入对象的钩子将在组件自身钩子之前调用。

可以自定义合并规则

React.create定义的组件

合并规则

  • 除了生命周期方法可以重复（优先组件里面的），其他地方的方法都不允许重复，否则会报错
  • 引入多个mixins，生命周期方法都会执行，执行顺序会按照mixins引入的顺序，然后再执行组件的

ps: 一本很少用React.createClass()方法来创建组件， 一般都是采用JSX + ES6

React.Component定义的组件

使用ES6编写的React组件时，不支持React mixins。而且，他们不会在React中支持ES6类。原因是它们被认为是有害的 。

react不赞同使用mixinx方法，主要原因：

  • ES6 classes不支持Mixins
  • Mixins将会修改state，所以开发者无法直接的确定state来自哪里，如果使用多个Mixins更甚
  • 如果使用多个Mixins时，他们设定或修改state将会造成命名冲突

react推荐使用高阶组件（Higher-Order Components）

POST常见的提交数据的方式

发表于 2019-06-20

HTTP POST 方式发送数据给服务器，请求主体的类型由Content-Type指定。在响应中，Content-Type标头告诉客户端实际返回的内容的内容类型

application/x-www-form-urlencoded

一般通过HTML form提交的POST请求，在不设置enctype属性的情况下，最终会以application/x-www-form-urlencoded方式提交数据。提交的数据按照 key1=val1&key2=val2 的方式进行编码。
